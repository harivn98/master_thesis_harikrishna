A.2. SAM 2 Segmentation: Six-Point Geometric Seed Grid
================================================================================

The six positive seed points used to initialise SAM 2.1 Hiera-Large video predictor on the first frame of each video. All coordinates are expressed as proportions of frame width and height, anchored in the bottom half of the frame.

Source file: segmentation/sam2/process_videos_sam2.py, lines 573–583.

    points = np.array([
        [width * 0.25, height * 0.80],   # Left road
        [width * 0.50, height * 0.85],   # Centre-bottom road
        [width * 0.75, height * 0.80],   # Right road
        [width * 0.50, height * 0.65],   # Upper road boundary
        [width * 0.35, height * 0.75],   # Additional left coverage
        [width * 0.65, height * 0.75],   # Additional right coverage
    ], dtype=np.float32)

    labels = np.array([1, 1, 1, 1, 1, 1], np.int32)  # All positive clicks

For input frames of 1920 × 1080 pixels, the absolute pixel coordinates are:

    Point   Position                 x (px)    y (px)
    -----   ----------------------   ------    ------
    1       Left road                  480       864
    2       Centre-bottom road         960       918
    3       Right road                1440       864
    4       Upper road boundary        960       702
    5       Additional left            672       810
    6       Additional right          1248       810

Post-hoc mask validation reference point (must lie inside the accepted mask):

    ref_point = (width // 2, int(height * 0.85))
    → (960, 918) for 1920 × 1080

Source: process_videos_sam2.py, line 630.

A.4. VLM Evaluation Prompt for Caption Refinement
================================================================================

The exact prompts sent to Qwen 2.5-VL-7B (cf. Bai et al., 2025) during the iterative caption refinement loop. The VLM receives a three-panel composite image (original with mask overlay, binary mask, inpainted result) alongside these instructions.

Source file: generation/VideoPainter/infer/edit_bench.py, lines 447–500.


A.4.1. System Prompt

    "You are a strict visual quality evaluator for image inpainting edits in
    driving scenes. Judge whether the edited image satisfies the instruction
    inside the masked region, and whether it remains visually consistent with
    the unmasked context. IMPORTANT: the input videos may have no centerline,
    a single centerline, or a double centerline. Evaluate based on the
    *desired post-edit result* implied by the instruction and the visible
    scene geometry.

    CRITICAL VALIDATION:
    - If instruction says SINGLE, REJECT captions describing DOUBLE (two
      parallel) lines
    - If instruction says DOUBLE, REJECT captions describing SINGLE line
    - If instruction says SOLID/CONTINUOUS, REJECT captions describing
      DASHED/INTERMITTENT
    - If instruction says DASHED/INTERMITTENT, REJECT captions describing
      SOLID/CONTINUOUS
    - If instruction specifies color (white/yellow), caption color must
      match exactly
    - Reject captions that imply existing markings that are not present in
      the original panel
    - Reject captions that are too vague (e.g., 'lane line')
    - Captions must include visual details (paint wear, texture, lighting,
      perspective alignment, placement)

    When revising captions, preserve ALL key descriptors from the instruction
    (single/double, solid/dashed, color, etc.)"


A.4.2. User Prompt Template

    "You will be shown a 3-panel image: (1) original with mask overlay (red
    indicates inpaint region), (2) the binary mask (white=inpaint), (3) the
    inpainted result.

    Editing instruction: {instruction}
    Current inpainting caption (used for the masked region): {current_caption}

    Return ONLY a JSON object with these keys:
    - verdict: "PASS" or "FAIL"
    - revised_caption: a better caption to try next if FAIL (<= 20 words).
      If PASS, you can repeat the current caption.
    - notes: short reason, mention any artifacts or mismatches.

    Rules for revised_caption (IMPORTANT):
    1. Must include key descriptors from the instruction (single/double,
       solid/dashed, color)
    2. Must include visual details about paint, texture, lighting
    3. Must reference placement and perspective alignment
    4. Must be <= 20 words
    5. Must not contradict the editing instruction
    6. Must not describe elements outside the masked region
    7. Must be more specific than the previous caption if FAIL"

A.6. LoRA Training Data Generation: Qwen VLM Captioning Prompt
================================================================================

The exact prompts sent to Qwen 2.5-VL-7B-Instruct for automated lane marking classification during LoRA training data generation. The model receives a cropped road region image (extracted via SAM 2 single-frame predictor mode).

Source file: segmentation/sam2/data_generation.py, lines 1062–1079.


A.6.1. System Prompt

    "You are a driving scene labeling assistant. You only describe lane line
    markings on the road surface. First, check whether the segmented region
    shows a CLEAR road surface with lane line markings clearly visible. If
    the road is not clear or the lane line markings are not clearly visible,
    output 'unknown' for ALL fields. Return ONLY valid JSON with keys:
    lane_marking_count, lane_marking_pattern, lane_marking_color. Do NOT wrap
    the JSON in Markdown, code fences, or backticks. Use this fixed vocabulary:
    lane_marking_count={single|double|unknown},
    lane_marking_pattern={solid|dashed|mixed|unknown},
    lane_marking_color={white|yellow|mixed|unknown}."


A.6.2. User Prompt

    "Look only at the segmented road region. Only answer if it is a clear
    road with lane line markings visible. Classify lane marking count (single
    vs double line), pattern (solid vs dashed), and color (white vs yellow)."
